# Contributing to the Project

First, thank you for considering contributing to our project! Your help is greatly appreciated. Below are some guidelines to help you get started.

## Development

To ensure a smooth and secure development process, please adhere to the following principles:

### 1. Tiered Transaction Limits (KYC/AML)

All transactions that involve the movement of funds must be protected by our Tiered Transaction Limit system. This is a critical component of our KYC/AML compliance strategy.

### 2. Two-Factor Authentication (2FA)

To enhance user security, we require all accounts to be protected by Two-Factor Authentication (2FA). When creating new features that require authentication, please ensure that they are compatible with our existing 2FA system.

### 3. API Rate Limiting

To protect our services from abuse, all API endpoints must be subject to rate limiting. This helps to prevent denial-of-service attacks and ensures a stable experience for all users.

### 4. Comprehensive Audit Logging

All significant user and admin actions must be logged. This includes, but is not limited to, the following:

- User login/logout
- Password changes
- Transaction history
- Admin actions (e.g., user management, system configuration)

### 5. Security Audit Trail

A user-facing security audit trail must be available to all users. This allows users to monitor their account activity and report any suspicious behavior.

## Submitting Changes

When you are ready to submit your changes, please create a pull request with a clear and concise description of the changes you have made. Be sure to reference any relevant issues or documentation.

Thank you for your contributions!
